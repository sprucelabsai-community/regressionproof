The Three Laws of TDD

We take the discipline seriously. These laws keep the feedback loop tight and the quality signal clean.

01

Fail First

Write no production code unless a failing test requires it.

02

Smallest Test

Write no more of a test than is sufficient to fail.

03

Smallest Code

Write no more production code than is sufficient to pass.
